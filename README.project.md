# Project

This project was created using the **Next.js Docker Development Template**.

It provides a complete local development environment running inside Docker.

---

# Development Stack

The environment may include the following services (depending on options selected during setup):

* Next.js
* Docker
* PostgreSQL
* Redis
* pgAdmin
* Redis Commander

All services run inside Docker containers so **no database or cache software needs to be installed on your system**.

---

# Start Development

Start the development environment:

```bash
./dev.sh start
```

---

# Rebuild Containers

If dependencies change or containers fail:

```bash
./dev.sh rebuild
```

---

# Stop the Environment

Stop all running containers:

```bash
./dev.sh stop
```

---

# View Logs

Follow container logs in real time:

```bash
./dev.sh logs
```

---

# Check Running Services

Ports may change automatically if default ports are already in use.

To see the correct URLs for your environment run:

```bash
./dev.sh ports
```

This command prints the URLs for:

* Web application
* pgAdmin (if enabled)
* Redis Commander (if enabled)

---

# Environment Variables

Project configuration is stored in:

```
.env
```

This file is automatically generated during setup.

It includes service ports, database credentials, and other environment configuration.

---

# Reset the Environment

If containers break or databases need resetting:

```bash
./dev.sh reset
```

Then rebuild the environment:

```bash
./dev.sh rebuild
```

---

# Project Structure

Key files and folders:

```
.dev.sh
setup.sh
.docker/
.env
src/
```

Docker configuration lives inside the `.docker` directory and development commands are managed through `dev.sh`.

---

# Notes

* All services run inside Docker containers.
* The development environment is isolated from your system.
* Ports are automatically assigned if defaults are already in use.
* Optional services such as PostgreSQL, Redis, pgAdmin, and Redis Commander may not be present if they were disabled during setup.
